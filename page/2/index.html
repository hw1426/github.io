jmeter函数-input篇

发表于 2018-05-11 | 分类于 自动化测试 ， Jmeter

本章节函数类型为input，此类函数可读取TXT、CSV、XML、DAT格式文件，常用于测试数据 （文件常放在/bin目录下？）

函数：__StringFromFile

  • 作用： Read a line from a file 文件中读取一行数据

  • 详细解释： StringFromFile函数可用于从文本文件中读取字符串。 这对于运行需要大量可变数据的测试非常有用。

    例如，在测试银行应用程序时，可能需要100或1000个不同的帐号。另请参阅可能更易于使用的CSV数据集配置测试元素。 但是，那目前不支持多个输入文件。每次调用它时，都会从文件中读取下一行。 所有的线程共享同一个实例，所以不同的线程会得到不同的线。 当到达文件末尾时，除非达到最大循环次数，否则将从头开始重新读取。

    如果在测试脚本中有多个对该函数的引用，则每个文件都将独立打开该文件，即使文件名相同。 [如果该值要在其他地方再次使用，请为每个函数调用使用不同的变量名称。]

    函数实例在线程之间共享，并且由任何需要下一行输入的线程打开（重新）打开文件，因此使用threadNumber作为文件名的一部分会导致不可预知的行为。

  • 特点：

    1. 数据文件支持多格式文件（.dat , .txt , .csv）；

    2. 数据文件和脚本放在 同一文件夹下（没实现，待验证），绝对路径已验证。

      PS.更正相对路径已验证，数据文件应放在jmeter/bin路径下

Jmeter函数-information篇

发表于 2018-05-10 | 分类于 自动化测试 ， Jmeter

本章节为information函数类型，多用于打印log调试用。

PS.补充官方Parameters和Examples

函数：__threadNum

  • 作用： 获取当前线程（用户）编号
  • 场景案例： 线程1录入user1 input ;线程2录入user2 input;……………
    在sampler中的parameters录入user${__threadNum} input
    适用场景在log中调试用

函数：__samplerName

  • 作用： 返回当前sampler的名称
  • 参数：
Attribute Description Required
Variable Name A reference name - refName - for reusing the value created by this function. Stored values are of the form ${refName}. No
  • 例子： ${__samplerName()}
  • 场景案例： 也是用于log中调试

Linux用户配置sudo权限

发表于 2018-04-26 | 分类于 Linux

在套路云上折腾docker中发现本地用户没有sudo权限（貌似本地虚拟机没有出现这个问题），度娘后总结如下：

sudo的工作过程：

  • 当用户执行sudo，系统优先寻找/etc/sudoers文件（不是sudoers.d文件夹）,判断用户是否有执行sudo的权限；
    未有权限报错：xxx is not in the sudoers file.This incident will be reported
  • 确认用户（用户组）具有可执行sudo权限后，让用户输入密码（或免密码）执行。

解决过程如下：
root用户登录,编辑sudoers文件
vim /etc/sudoers

在# User privilege specification 下插入（可选）

第一行:允许用户youuser执行sudo命令(需要输入密码).

第二行:允许用户组youuser里面的用户执行sudo命令(需要输入密码).

第三行:允许用户youuser执行sudo命令,并且在执行的时候不输入密码.

第四行:允许用户组youuser里面的用户执行sudo命令,并且在执行的时候不输入密码。

%表示用户组

这样普通用户有sudo权限
