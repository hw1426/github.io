Jmeter函数-information篇

发表于 2018-05-10   |  阅读量 次 | 分类于 自动化测试 ， Jmeter

本章节为information函数类型，多用于打印log调试用。

函数：__threadNum

  • 作用： 获取当前线程（用户）编号
  • 场景案例： 线程1录入user1 input ;线程2录入user2 input;……………
    在sampler中的parameters录入user${__threadNum} input
    适用场景在log中调试用

函数：__samplerName

  • 作用： 返回当前sampler的名称
  • 例子： ${__samplerName()}
  • 场景案例： 也是用于log中调试

    函数：__macheineIP

  • 作用： 函数返回本地IP地址
  • 例子： ${__machineIP(#### )} 和${__machineIP} 都返回- 本机IP
  • 场景案例： 1、还是log- 2、分布式

函数：__machineName

  • 作用： 函数返回本地机器的名称
  • 例子： ${__machineName()}和${__machineName} 都返回hostname
  • 场景案例： 还是log

函数：__time

  • 作用： 返回各种格式的当前时间

  • 格式如下：

  • 例子：

    ${__time(dd/MM/yyyy,)}return 21/01/2018

    ${__time(YMD,)} return20180121

    ${__time()} return 当前时间戳

    还有其他表达方式:

    ${__time(EEE\, dd MMM yyyy)}return Thu,10 May 2018

函数：__log

  • 作用： 在log中打印

  • 场景实现：
    在前置、后置处理器及线程组中可以打印log，输出在log和view Results tree

  • log和view Results tree的区别
    log中可以持久化，viewResults Tree 运行线程组会覆盖当前信息

函数：__logn

  • 作用： 和log相似，但只输出在log里
